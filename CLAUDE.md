# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start        # run the bot
```

No test suite or linter is configured.

## Environment variables

Copy `.env.example` to `.env` and fill in all values:

| Variable | Purpose |
|---|---|
| `SLACK_BOT_TOKEN` | Slack bot OAuth token (`xoxb-`) |
| `SLACK_SIGNING_SECRET` | Slack app signing secret |
| `SLACK_APP_TOKEN` | Socket Mode app token (`xapp-`) |
| `OPENAI_API_KEY` | GPT-4o candidate matching |
| `HR_CHANNEL_ID` | Slack channel ID where HR notifications are posted |
| `ATS_API_URL` | REST endpoint returning open vacancies as JSON array |
| `GOOGLE_SHEET_ID` | Google Spreadsheet ID for referral log |
| `GOOGLE_SERVICE_ACCOUNT_EMAIL` | Service account email with Sheets access |
| `GOOGLE_PRIVATE_KEY` | Service account private key (newlines as `\n`) |

## Architecture

The bot runs in **Slack Socket Mode** (no public HTTP endpoint needed). Entry point is `src/app.js`.

### Request flow

1. User types `/refer` in Slack → `src/commands/refer.js` opens the `referral_modal`.
2. User submits the modal → `src/handlers/viewSubmission.js` validates fields, then fires an async pipeline (non-blocking, ack is sent immediately):
   - `atsService.getVacancies()` — fetches open vacancies from the ATS REST API (5-min in-memory cache).
   - `claudeService.matchCandidate()` — sends candidate profile + vacancies to GPT-4o; returns structured JSON with `matched`, `vacancy_id`, `vacancy_title`, `vacancy_url`, `match_score`, `summary`, `match_reason`.
   - `sheetsService.appendReferral()` — appends a row to the `Referrals` sheet (auto-creates headers on first write).
   - `slackService.notifyHR()` — posts a Block Kit message to `HR_CHANNEL_ID`.
   - `slackService.sendConfirmationDM()` — DMs the referrer with the outcome.

### Key conventions

- `vacancy_url` is constructed in code (`https://hellowehire.com/positions-user/{id}/candidates`) after parsing the GPT response, not generated by the model.
- GPT-4o is called with `response_format: { type: 'json_object' }` — no need to strip markdown fences from the response.
- ATS vacancy fields `requirements`, `responsibilities`, and `description` are HTML-stripped and truncated to 600 chars before being sent to GPT.
- Match threshold: `match_score >= 60` is considered a match.
- Google Sheets auth uses a JWT service account; the private key `\n` sequences must be unescaped (done in `sheetsService.js`).
